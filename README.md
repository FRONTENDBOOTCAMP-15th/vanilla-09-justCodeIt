## ✨ 프로젝트 주제, 목적 및 채택 사유

### 주제

이 프로젝트는 나이키 공식 온라인 스토어 UI/UX를 참고하여, 반응형 웹 기반의 쇼핑몰 서비스를 직접 구현하는 것을 목표로 한 웹 프론트엔드 프로젝트이다.
단순한 정적 페이지를 만드는 것이 아니라,

- 로그인/회원 인증
- 장바구니
- 위시리스트
- 제품 상세 페이지
- 추천 상품 컴포넌트

등 실제 전자상거래 웹사이트와 동일한 기능을 바닐라 JS + 타입스크립트 + TailwindCSS로 직접 구현한 학습 프로젝트이다.
<br/>

### 목적

1. 실제 서비스 수준의 웹 서비스 구조 이해
2. API 기반의 동적 데이터 렌더링 경험
3. 타입스크립트 기반의 안정적인 개발 연습
4. 반응형 UI/UX 설계 능력 향상
5. 컴포넌트 기반 재사용성 설계 학습
6. 협업을 대비한 Git 브랜치 전략 및 코드 관리 능력 습득
   <br/>

### 채택 이유

쇼핑몰은 프론트엔드에서 구현해야 하는 거의 모든 핵심기능을 포함하기 때문에 쇼핑몰 CRUD의 흐름을 전체적으로 경험하고 싶었고,
모바일에서 데스크탑으로 넘어가는 구간에서 이미지 위치, 텍스트 정렬, 버튼 UI가 달라지는 구조를 반응형으로 구현해보고 싶어서 이 프로젝트를 채택하였습니다.

## 💪 협업 방식

### ✔ 챌린지

- 데일리 스크럼을 통해 정한 오늘의 할일을 완수하기
- 효율적인 시간관리

## 회고 및 느낀점

## 🧑‍🎨 김지유

### **잘된 점**

- 반응형 UI 구현
- 조건별 제품 API 호출
- 추천 제품 컴포넌트화로 재사용성 확보

### **아쉬운 점 & 개선**

- 모바일 필터 기능 미구현
- 필터 옵션을 다양하게 확장해볼 계획

### **트러블슈팅**

- 추천 제품 슬라이드가 넘어가지 않는 문제  
  → `innerWidth`, `currentIndex`, `slice` 조합으로 슬라이드 정상 구현

---

## 😇 유홍상

### **잘된 점**

- 담당 페이지 UI 및 주요 기능 안정적으로 구현
- Web Components로 공통 헤더/푸터 제작해 재사용성 향상

### **아쉬운 점 & 개선**

- 장바구니 아이콘 상품 카운터 기능 미구현
- 이미지 로딩 최적화 필요
- 위로가기 버튼 기능 추가 예정

### **트러블슈팅**

- 상세페이지 위시리스트 상태가 새로고침 시 유지되지 않는 문제  
  → 페이지 진입 시 서버에서 위시리스트 목록 재조회하도록 수정

## 🐧 허예진

### **잘된 점**

- 모바일 퍼스트 & 반응형 레이아웃 구조 명확히 준수
- 위시리스트 삭제 기능 구현
- 모바일 헤더 검색 애니메이션 구현

### **아쉬운 점 & 개선**

- 홈 이동 시 FOUC(스타일 지연 적용) 문제 발생
- 슬라이더 반응형 개선 필요  
  → SPA 형태로 전환 시 더 부드러운 화면 전환 가능

### **트러블슈팅**

- TypeScript null 타입 오류 반복 발생  
  → 조건문 분리 + 타입 narrowing으로 해결

## 🌝 박지은

### **잘된 점**

- 모바일/데스크탑 반응형 UI 구현
- 장바구니–위시리스트 연동
- 장바구니 수량 변경·삭제 기능, 전체 비우기까지 완성도 있게 구현
- 주문 완료 시 UI 갱신 흐름 자연스럽게 처리

### **아쉬운 점 & 개선**

- 장바구니/위시리스트 UI 로직이 길어져 리팩터링 필요
- 사이즈 선택 시 품절 표시 등 UI 개선 가능

### **트러블슈팅**

- 장바구니 북마크 API 422 오류 → 스펙에 맞게 body의 id 수정
- 장바구니 위시리스트 즉시 반영되지 않는 문제 → 북마크 변경 후 위시리스트 재조회

## 💬 프로젝트를 마치며 성장한 경험

👨‍💻 김지유

- 바닐라 프로젝트에서 처음으로 TypeScript를 사용하며 타입 지정과 구조에 대한 이해도가 높아짐
- 여러 params 조건을 조합해 API를 연결하면서 복잡한 요청 처리를 경험함
- TailwindCSS를 활용해 빠르고 효율적으로 스타일링
- Netlify 배포까지 진행하며 프로젝트 완성도를 높임

👨‍💻 유홍상

- 처음으로 TypeScript와 Axios를 사용해 API를 연동해 보며, 서버와의 통신 구조와 비동기 처리 방식을 직접 이해하게 되었고, 그 과정에서 프론트엔드 개발에 대한 이해도가 높아짐.

👨‍💻 허예진

- UI 개발의 전체 흐름을 스스로 설계
- TailwindCSS를 실전에서 사용
- 타입 오류 상황 경험 및 해결

👩‍💻 박지은

- 타입스크립트로 완성도 있는 결과물을 도출한 첫 프로젝트
- API 스펙과 타입 불일치 문제에서 오류를 해결해보면서 API 명세 보는 법, 타입 정의의 필요성을 체감
